# System Design Document — Agent 1C PoC

## 1. Обзор системы

Agent 1C — это агентная система для безопасного анализа данных 1С через естественный язык с автоматической анонимизацией персональных данных.

### 1.1 Ключевые возможности

- Обработка запросов на естественном языке → генерация запросов 1С
- Анонимизация ПнД (ФИО, терминалы, причины) **до** и **после** выполнения
- Автоматическое исправление ошибочных запросов (retry loop)
- Сохранение результатов с возможностью последующего анализа 
- Guardrails: валидация запросов, защита от инъекций, лимиты попыток

---

## 2. Архитектурные решения

| Решение | Описание | Обоснование |
|---------|----------|-------------|
| **LangGraph State Machine** | Агент реализован как граф состояний | Явный control flow, возможность ветвления, отладка |
| **Mistral API (Cloud LLM)** | mistral-large-latest для генерации запросов 1С | Качество генерации, поддержка русского языка |
| **COM-соединение с 1С** | win32com.client для прямого подключения | Нативная интеграция, отсутствие промежуточных API |
| **Двухуровневая анонимизация** | Санитизация запроса + анонимизация результатов | Защита ПнД на входе и выходе |
| **Query Validator** | Статическая валидация перед выполнением | Защита от инъекций, только SELECT-запросы |
| **Retry с ограничением** | MAX_FIX_ATTEMPTS=5, детекция циклов | Баланс между исправлением и бесконечным циклом |
| **File Storage с TTL** | Сохранение результатов (30 дней) | Повторный анализ, audit trail |

## 5. State / Memory / Context Handling

### 5.1 Agent State (LangGraph)

```python
class AgentState(TypedDict):
    user_query: str           # Исходный запрос
    sanitized_query: str      # Очищенный от ПнД
    generated_query: str      # Сгенерированный запрос 1С
    validation_passed: bool   # Флаг валидации
    validation_errors: List[str]
    execution_result: List    # Данные из 1С
    error: Optional[str]      # Текущая ошибка
    anonymized_data: List     # Анонимизированные данные
    html_table: str           # HTML для отображения
    analysis: str             # Текстовая сводка
    fix_attempts: int         # Счётчик попыток исправления
    max_attempts: int         # Лимит (5)
    date_range: tuple         # (start, end) дат
    seen_errors: List[str]    # История ошибок (детекция циклов)
    success: bool             # Флаг успеха
```

### 5.2 Memory Policy

| Тип | Хранение | TTL |
|-----|----------|-----|
| **Session State** | In-memory (AgentState) | До конца запроса |
| **File Storage** | Disk (`data/anonymized/`, `data/raw/`) | 30 дней |
| **COM Connection** | Singleton (глобальное) | До остановки сервера |

### 5.3 Context Budget

- **LLM Input:** SYSTEM_PROMPT (~500 токенов) + user query
- **LLM Output:** Max 500 токенов (запрос 1С)
- **Temperature:** 0.1 (детерминированная генерация)

## 6. Retrieval-контур

### 6.1 Источники данных

| Источник | Тип | Описание |
|----------|-----|----------|
| **1С:Производство** | COM | Документ |
| **File Storage** | JSON/CSV | Сохранённые результаты запросов |

### 6.2 Индексация и поиск

В PoC-версии **отсутствует** классический retrieval (RAG). Поиск заменяется:

1. **Date Range Detection** — автоматическое определение периода из запроса
2. **Query Generation** — LLM генерирует точный запрос к 1С

## 8. Failure Modes, Fallback, Guardrails

### 8.1 Типы отказов

| Тип | Причина | Обработка |
|-----|---------|-----------|
| **LLM Error** | API unavailable, rate limit | Возврат ошибки пользователю |
| **COM Error** | 1С недоступен, таймаут | Возврат ошибки, retry не применяется |
| **Query Syntax Error** | Ошибка в запросе 1С | Auto-fix (до 5 попыток) |
| **Validation Error** | Запрещённые слова | Блокировка до генерации |
| **Anonymization Error** | Нет данных | Пустой результат с сообщением |
| **Cycle Detection** | Повторяющиеся ошибки | Прерывание цикла после 2 повторов |

### 8.2 Guardrails

| Защита | Реализация |
|--------|------------|
| **SQL Injection** | Только ВЫБРАТЬ, blacklist ключевых слов |
| **ПнД утечка** | Двухуровневая анонимизация (вход + выход) |
| **Infinite Loop** | MAX_FIX_ATTEMPTS=5, seen_errors детекция |
| **Resource Exhaustion** | COM_MAX_ROWS=1000, FILE_TTL_DAYS=30 |
| **Admin Access** | ADMIN_API_KEY для чувствительных endpoints |

### 8.3 Fallback Strategy

```
LLM недоступен → Ошибка "Сервис временно недоступен"
1С недоступен → Ошибка "Соединение с 1С разорвано"
Query не выполняется → Auto-fix (5 попыток) → Ошибка пользователю
Валидация не прошла → Ошибка до выполнения
```

## 9. Технические и операционные ограничения

### 9.1 Latency

| Операция | Ожидаемое время | SLA |
|----------|-----------------|-----|
| **LLM Generation** | 2-5 сек | < 10 сек |
| **COM Query** | 1-2 сек (первое подключение занимает ~2-3 мин) | < 30 сек |
| **Anonymization** | < 1 сек (1000 строк) | < 2 сек |
| **Total End-to-End** | 5-20 сек | < 60 сек |

### 9.2 Cost

| Ресурс | Стоимость | Лимит |
|--------|-----------|-------|
| **Mistral API** | бесплатно | частота запросов в минуту |
| **1С COM** | Бесплатно (локально) | - |
| **Storage** | Локальный диск | 30 GB |

### 9.3 Reliability

| Компонент | Availability | Notes |
|-----------|--------------|-------|
| **FastAPI** | 99.9% (локально) | Зависит от хоста |
| **Mistral API** | 99.5% (SLA провайдера) | Внешняя зависимость |
| **1С COM** | Зависит от ИТ-инфраструктуры | Критическая зависимость |
