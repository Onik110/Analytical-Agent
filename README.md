# Agent analyst: Агент-аналитик производственного оборудования

## Проблематика
Анализ эффективности станочного оборудования в производственной среде сегодня превратился в многоуровневую рутину, где простые вопросы требуют значительных временных затрат. Оперативные запросы заставляют пользователя вручную сводить данные из разных мест, после чего самостоятельно калькулировать ключевые метрики. На такой анализ уходит 15–20 минут, в течение которых специалист не управляет производством, а лишь ищет информацию. Сложные задачи — сравнительная аналитика по цеху, выявление корреляций между простоем и логистикой заготовок, прогнозирование загруженности на неделю вперёд — вообще не решаются оперативно: для их выполнения приходится обращаться к программисту, который уже перегружен потоком запросов на составление отчётов от разных подразделений. Задача встаёт в очередь на часы или даже дни, а к моменту получения отчёта исходная проблема либо усугубляется, либо становится неактуальной. 

## Задача
Автоматизация анализа эффективности станочного оборудования через запросы на естественном языке. Агент извлекает данные из БД, переводит сырые показатели в бизнес-метрики (КПВ, время выпуска изделия, загруженность) и выявляет узкие места в производстве.

## Аудитория:
1. Начальники производственных участков — контролируют выполнение плана, распределяют загрузку станков
2. Специалисты по планированию — формируют производственный план с учётом реальной загрузки оборудования
3. Технологи/механики производства — анализируют эффективность оборудования, ищут причины простоев


## Что именно сделает PoC на демо:
Пользователь задает вопрос на естественном языке → агент:
  - Извлекает данные из 1С через COMConnector или OData
  - Переводит сырые данные в бизнес-метрики (КПВ, загруженность, время выпуска)
  - Выявляет аномалии и дает рекомендации
  - Формирует структурированный ответ с инсайтами

## Что НЕ делает PoC, явные out-of-scope:
- Изменение данных в 1С — только режим чтения; запрещены создание, редактирование и удаление документов
- Управление оборудованием — агент не запускает и не останавливает станки, не взаимодействует с системами АСУ ТП
- Интеграция с внешними системами — не подключаются MES, SCADA, календари или другие ERP; источник данных — только 1С:ERP
- Многопользовательский RBAC — для демо один пользователь с правами «только чтение»; нет ролевой модели доступа
