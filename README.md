# 1С Ассистент — AI-помощник для анализа данных 1С

Интеллектуальная система анализа данных 1С с использованием LangGraph и Mistral AI. Автоматически генерирует запросы к 1С, anonymзирует персональные данные и предоставляет результаты в удобном веб-интерфейсе.

## Возможности

- **Автоматическая генерация запросов 1С** — опишите, что нужно, на естественном языке
- **Защита персональных данных** — автоматическая анонимизация ФИО и чувствительных данных
- **Умная валидация** — проверка и исправление запросов перед выполнением
- **COM-соединение с 1С** — прямое подключение к базе данных
- **Веб-интерфейс** — удобный интерфейс для работы с системой

## Требования

- **Windows 10/11** (COM работает только на Windows)
- **Python 3.10+**
- **Установленный клиент 1С:Предприятие 8.3** (с COM-соединением)
- **Mistral API ключ** — получить на https://mistral.ai
- Доступ к базе данных 1С

## Установка

### 1. Настройка окружения
```bash
# Скопируйте пример конфигурации
copy .env.example .env

# Отредактируйте .env, указав ваши параметры:
# - COM_SERVER, COM_BASE, COM_USER, COM_PASSWORD — подключение к 1С
# - MISTRAL_API_KEY — API ключ Mistral AI или другой LLM
```

### 2. Установка зависимостей
```bash
# Создаем виртуальное окружение
python -m venv venv

# Активируем
venv\Scripts\activate

# Устанавливаем зависимости
pip install -r requirements.txt
```

## Запуск

### Запуск backend (FastAPI):
```bash
venv\Scripts\activate
python -m uvicorn backend.main:app --host 0.0.0.0 --port 8000 --reload
```

Backend будет доступен на:
- API: http://localhost:8000
- Документация: http://localhost:8000/docs

### Запуск frontend (React + Vite):
```bash
cd frontend
npm install
npm run dev
```

Frontend будет доступен на: http://localhost:5173

## Конфигурация

### Переменные окружения (.env)

#### 1С COM Connection:
```env
COM_SERVER=server_name       # Сервер 1С
COM_PORT=port                # Порт 1С
COM_BASE=db_name             # База данных
COM_USER=user                # Пользователь
COM_PASSWORD=pass            # Пароль
COM_MAX_ROWS=1000            # Максимум строк в результате
COM_QUERY_TIMEOUT=30         # Таймаут запроса (секунды)       
```

#### Mistral AI:
```env
MISTRAL_API_KEY=ваш_ключ      # API ключ Mistral
MISTRAL_MODEL=mistral-large-latest  # Используемая в проекте модель
```

#### Сервер:
```env
FASTAPI_HOST=0.0.0.0
FASTAPI_PORT=8000
DEBUG_MODE=True
```

## API Endpoints

- `POST /api/query` — обработать запрос к 1С
- `GET /api/health` — проверка состояния системы
- `GET /api/metrics` — метрики безопасности
- `GET /docs` — Swagger документация

### Тестовые запросы

В файле **`docs/test-queries.md`** приведены примеры 16 тестовых запросов разной сложности:


- Простые запросы (терминалы, ткачи)
- Запросы с конкретными датами
- Агрегации и статистика
- Причины простоев
- Сложные аналитические запросы
- Edge-кейсы
- Запросы для проверки безопасности

Тестирование производилось на **100 тестовых запросах**

### Запуск тестов

**Все запросы:**
```powershell
python -m backend.services.run_tests --all
```

**Конкретные запросы:**
```powershell
python -m backend.services.run_tests --queries 1,2,3,10,15
```

**По категории:**
```powershell
python -m backend.services.run_tests --category "Простые запросы"
```

**Сохранение отчета:**
```powershell
python -m backend.services.run_tests --all --save my_test.json
```
