# Обоснование идеи

## Прикладная задача

На производственном предприятии с большим количеством станков технологи и начальники цехов тратят 15–20 минут на каждый запрос об эффективности оборудования, калькулировать метрики в уме или Excel (КПВ, время выпуска), искать закономерности в истории. Сложные задачи (прогнозирование загруженности, выявление корреляций) уходят в очередь на часы или дни. Система не выявляет автоматически повторяющиеся проблемы — например, еженедельные простои из-за логистики заготовок.
Агент-аналитик решает задачу за несколько секунд: принимает вопрос на естественном языке, извлекает данные из 1С через COMConnector/OData, рассчитывает бизнес-метрики, выявляет аномалии через сравнение с историей и возвращает структурированный ответ с инсайтами и рекомендациями — без участия программиста и без ручного поиска через запросы.

## Потенциальные сценарии использования

### Основные сценарии использования 
1. Диагностика простоя — пользователь спрашивает о причинах остановки конкретного станка за период. Агент извлекает необходимые данные, после чего рассчитывает КПВ, время выпуска и агрегирует причины простоев.
2. Прогнозирование загруженности — запрос на будущий период.
3. Выявление узких мест — анализ системных проблем по предприятию. Агент собирает данные по всем станкам цеха, агрегирует историю простоев по причинам и выявляет корреляции (например, 75% простоев по заготовкам происходят в 12:00–13:00, что совпадает с перерывом рабочего).
4. Оптимизация плана — запрос на улучшение эффективности. Агент анализирует историю простоев по категориям (ТО, заготовки, наладка), рассчитывает оптимальный порядок заданий для минимизации переналадок и генерирует рекомендации по логистике и расписанию ТО.

### Edge-кейсы 

- Запрос затрагивает несколько объектов 1С — агент выбирает наиболее релевантные, результат может быть неполным.
- Запрос сформулирован неоднозначно — несколько станков с похожими названиями или не указан период — агент уточняет интерпретацию у пользователя перед выполнением.
- Справочники 1С содержат много объектов — агент итерирует по метаданным для поиска нужного станка, время ответа может превышать целевой показатель при большом объёме данных.

## Ограничения

### Технические:
- Время ответа (p95): < 15 секунд 
- Доступность агента: 99.5%
- Требования к инфраструктуре: Windows Server, 8 ГБ RAM, установленный клиент 1С для COMConnector
- Ограничение: 1 запрос/5 сек на пользователя (защита от перегрузки 1С)

### Операционные:
- Срок разработки PoC: 3 недели + 1 неделя тестирования
- Поддержка: доступ к 1С-программисту 2–3 часа/неделю для валидации запросов
- Глубина анализа: данные за последний год

## Архитектурный набросок

Модули агента:
* Router — извлечение сущностей (станок, период) и определение типа запроса
* Analyzer — построение последовательности запросов к 1С и расчёт метрик
* State & Memory — хранение контекста диалога (последний станок, период)
* KPI Calculator — расчёт бизнес-метрик (КПВ, время выпуска, загруженность)
* Recommender — генерация практических рекомендаций на основе анализа

Интеграции:
* COMConnector/OData — основные методы подключения к 1C
* Query Validator — валидация параметров перед выполнением (защита от инъекций)
